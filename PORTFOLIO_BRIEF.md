## 6. Design Requirements

### Style & Aesthetic
- **Modern, minimal, clean** — think Vercel, Linear, Stripe, or Anthropic's website style
- **Dark mode by default** with light mode toggle (use Tailwind dark mode)
- **Generous whitespace** — don't cram content
- **Smooth scroll behavior** between sections
- **Subtle animations** on scroll (use Framer Motion) — but don't overdo it
- **Mobile-first responsive** — must look great on phone first, then desktop

### Color Palette (suggestions — pick one)
**Option A (Tech / AI vibe):**
- Background: `#0A0A0A` (dark) / `#FFFFFF` (light)
- Accent: `#3B82F6` (blue) or `#8B5CF6` (purple)
- Text: `#E5E5E5` / `#171717`

**Option B (Minimal Anthropic-inspired):**
- Background: `#FAFAF7` (warm white) / `#1A1A1A` (dark)
- Accent: `#D97757` (warm orange) or `#1F3864` (deep blue)
- Text: `#171717` / `#E5E5E5`

### Typography
- **Headings:** Inter, Geist, or Space Grotesk
- **Body:** Inter or system font stack
- **Code/Mono:** JetBrains Mono or Fira Code

## 7. Technical Requirements

### Stack
- **Framework:** Next.js 14+ (App Router) with TypeScript
- **Styling:** Tailwind CSS
- **Animations:** Framer Motion (subtle, not overwhelming)
- **Icons:** Lucide React + Simple Icons (for tech badges)
- **Theme:** next-themes for dark/light mode
- **Forms:** React Hook Form + Zod (if contact form is included)

### Performance
- Lighthouse score: 95+ on Performance, Accessibility, Best Practices, SEO
- Use Next.js Image component for any images
- Lazy-load below-the-fold content

- Proper meta tags (title, description, OG image)
- Structured data (JSON-LD) for Person schema
- Sitemap.xml and robots.txt
- Open Graph image for social sharing (1200x630)

### Accessibility
- Semantic HTML
- Keyboard navigation works everywhere
- ARIA labels where needed
- Color contrast meets WCAG AA
